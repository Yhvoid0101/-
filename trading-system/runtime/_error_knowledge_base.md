v564引入的`_simulate_with_time_stop`函数使用了错误的PnL计算公式:

```python
# v566 错误公式 (第564行):
net_pnl = pnl_pct * actual_risk / 0.018
# 化简: net_pnl = pnl_pct * risk_mult  (价格变动% × risk_mult)
```

而v559使用的是正确的风险导向公式:

```python
# v559 正确公式 (第441-446行):
risk_multiple = pnl_pct / stop_distance    # 风险倍数 (如4R, -2R)
adjusted_pnl = risk_multiple * actual_risk  # PnL = R倍数 × 仓位风险
net_pnl = adjusted_pnl - cost              # 减去手续费
```

### 实际数据验证 (第1笔交易 ADA_USDT)

|------|------------|------------|
| entry | 0.3334 | 0.3334 |
| exit | 0.3352 | 0.3352 |
| pnl_pct | -0.5313% | -0.5313% (相同) |
| actual_risk | 0.0216 (risk_mult=1.2, 反风控) | 0.018 (risk_mult=1.0, 无反风控) |
| stop_distance | 0.604% | - |
| risk_multiple | -0.88R | - |
| **net_pnl** | **-2.0%** (正确) | **-0.53%** (错误) |
| **差异** | **3.76倍** | |

### 核心错误分析

1. **v566的PnL = 价格变动% × risk_mult**: 这相当于满仓(100%仓位)的价格风险暴露,而非1.8%仓位风险
2. **v559的PnL = R倍数 × 仓位风险**: 标准的风险导向仓位管理
3. **关键差异**: v566漏掉了`/ stop_distance`这一步,把价格变动直接当作PnL

### 错误结论纠正

1. **ERR-034 (v564 15m共振过滤好信号)**: 根因分析部分错误。15m共振确实过滤信号,但PnL从+87.3%→+6.1%的主要原因是PnL公式错误,不是15m共振

3. **ERR-036 (v566 波动率regime破坏反风控正反馈)**: 根因分析错误。不是波动率regime MULT=0.5破坏正反馈,而是PnL公式错误导致月度PnL被低估,无法触发反风控


## ERR-20260629-039: v569月内PnL前置风控副作用 (正常月份降仓减少盈利)

1. **修改stop距离时必须考虑对TP距离的连锁影响** — stop_distance增大若TP也用stop_distance计算,则TP距离被动增大,小ATR交易难以达到TP
